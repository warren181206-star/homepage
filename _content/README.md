# 콘텐츠 관리 가이드

> 이 폴더는 홈페이지 콘텐츠를 MD 문서로 관리하기 위한 시스템입니다.

---

## 폴더 구조

```
_content/
├── README.md          # 이 파일 (가이드)
├── pages.md           # 전체 페이지 목록 및 상태
└── pages/             # 페이지별 MD 문서
    ├── 00_헤더.md
    ├── 01_푸터.md
    ├── 02_홈.md
    └── ...
```

---

## 작업 규칙

### 1. MD 문서 → HTML 작성

**MD 문서가 원본입니다.**

1. `_content/pages/` 폴더의 MD 문서를 먼저 수정
2. MD 내용을 기반으로 `pages/` 폴더의 HTML 작성
3. MD의 이미지 순서대로 HTML에 반영

### 2. HTML 먼저 수정한 경우

**반드시 MD에 미러링해야 합니다.**

1. HTML 수정 후 해당 MD 문서 열기
2. 변경된 내용을 MD에 반영
3. YAML의 `updated` 날짜 갱신

```yaml
---
updated: 2025-01-12 21:50  # ← 이 날짜를 현재 시간으로 수정
---
```

### 3. updated 날짜 관리

- 사용자가 MD 수정 시 직접 `updated` 갱신
- 사용자가 빠뜨린 경우 Claude가 체크하여 갱신
- 형식: `YYYY-MM-DD HH:MM`

## MD 문서 형식

### YAML 프론트매터 (필수)

```yaml
---
title: 페이지명
html: ../pages/05_consignment.html
url: /consignment
done: true
updated: 2025-01-12 21:55
---
```

| 필드 | 설명 |
|------|------|
| title | 페이지 제목 |
| html | 대응하는 HTML 파일 경로 |
| url | 아임웹 URL |
| done | 완성 여부 (true/false) - 옵시디언에서 체크박스로 표시 |
| updated | 마지막 수정 일시 |

## 콘텐츠 수정 방법

### 텍스트 수정

```markdown
# 섹션1: 메인 헤더

## 내용
제목: 안전하고 빠르게!     ← 이 텍스트 수정
설명: 금을 확보하는 방법
```

### 이미지 추가/변경

```markdown
## 내용
![[새이미지.jpg]]          ← 새 이미지 파일명으로 변경
```

→ 이미지는 `D:\Obsidian\_attachments\` 폴더에 저장

### 스타일 속성 수정

```markdown
## 속성
배경색: #1a1a1a            ← 색상 코드 변경
글자크기: PC 32px, 모바일 24px   ← 크기 변경
```

### 작업 요청하기

```markdown
## 요청사항
- 이 섹션의 배경을 더 어둡게 해주세요
- 제목 글씨를 금색(#b78e64)으로 변경해주세요
- 모바일에서 2열로 표시해주세요
```

## Claude 작업 시 규칙

1. **MD 우선**: 페이지 작업 시 MD 문서 먼저 확인/수정
2. **미러링 필수**: HTML만 수정한 경우 MD에 반영
3. **날짜 체크**: updated 필드 누락 시 자동 갱신
4. **이미지 순서**: MD의 이미지 배치 순서를 HTML에 그대로 반영
5. **요청사항 처리**: `## 요청사항` 내용 확인 후 HTML에 반영하고 완료 시 해당 항목 삭제
